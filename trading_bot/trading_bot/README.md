# Simplified Trading Bot — Binance Futures Testnet (USDT-M)

A small, structured Python CLI application that places MARKET, LIMIT, and
STOP_LIMIT orders on the Binance USDT-M **Futures Testnet**, with input
validation, structured logging, and clean error handling.

## Project Structure

```
trading_bot/
  bot/
    __init__.py
    client.py          # Binance API wrapper (only place that talks to Binance)
    orders.py           # Validates input, calls client, formats results
    validators.py        # Pure input validation, no I/O
    logging_config.py    # Rotating file + console logging setup
  tests/
    test_validators.py   # Unit tests for validation logic
  cli.py                 # argparse-based CLI entry point
  logs/
    trading_bot.log       # Sample log (includes 1 MARKET + 1 LIMIT order)
  requirements.txt
  .env.example
  README.md
```

## Setup

1. **Create a Binance Futures Testnet account** and generate API keys:
   - Go to https://testnet.binancefuture.com
   - Register / log in, then generate an API Key + Secret from the testnet
     dashboard.

2. **Clone this repo and install dependencies** (Python 3.9+):
   ```bash
   python -m venv venv
   source venv/bin/activate        # Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Configure credentials.** Copy `.env.example` to `.env` and fill in your
   testnet keys:
   ```bash
   cp .env.example .env
   ```
   ```
   BINANCE_API_KEY=your_testnet_api_key_here
   BINANCE_API_SECRET=your_testnet_api_secret_here
   ```
   Credentials are also read from real environment variables if you'd
   rather not use a `.env` file.

## How to Run

**Market order (BUY):**
```bash
python cli.py --symbol BTCUSDT --side BUY --type MARKET --quantity 0.01
```

**Limit order (SELL):**
```bash
python cli.py --symbol BTCUSDT --side SELL --type LIMIT --quantity 0.01 --price 65000
```

**Stop-limit order (bonus feature, BUY):**
```bash
python cli.py --symbol BTCUSDT --side BUY --type STOP_LIMIT --quantity 0.01 \
  --price 61000 --stop-price 60900
```

**Adjust console log verbosity** (file logging always captures DEBUG detail):
```bash
python cli.py --symbol BTCUSDT --side BUY --type MARKET --quantity 0.01 --log-level DEBUG
```

Every run prints:
- an order request summary (symbol, side, type, quantity, price if applicable)
- the order response (`orderId`, `status`, `executedQty`, `avgPrice`)
- a clear `SUCCESS`/`FAILURE` line

All requests, responses, and errors are also written to `logs/trading_bot.log`
(rotating, 5 MB × 3 backups).

## Running Tests

```bash
python -m pytest tests/ -v
```

Covers symbol/side/type/quantity/price validation, including the
LIMIT-requires-price and STOP_LIMIT-requires-stop-price rules.

## Error Handling

- **Invalid input** (bad symbol, side, order type, non-numeric or
  non-positive quantity/price, missing price for LIMIT orders, etc.) is
  caught by `bot/validators.py` before any network call is made, and
  reported as a clean `FAILURE` message — no stack trace shown to the user.
- **API errors** (e.g. insufficient testnet balance, invalid symbol on
  Binance's side, bad API key) are caught around the `python-binance` call
  in `bot/client.py`, logged, and surfaced as a `FAILURE` message.
- **Network failures** (timeouts, DNS errors, connection refused) are
  caught by the same handler in `bot/client.py` and reported the same way,
  so the CLI never crashes with an unhandled traceback.

## Assumptions

- **STOP_LIMIT** is implemented as Binance Futures' `STOP` order type
  (a stop-triggered limit order), since Binance Futures has no distinct
  "STOP_LIMIT" enum value — this was chosen as the bonus order type.
- Only USDT-M Futures Testnet is supported (as specified); Coin-M and Spot
  are out of scope.
- `timeInForce` defaults to `GTC` for LIMIT/STOP_LIMIT orders since the
  task didn't specify one and GTC is Binance's own default.
- The CLI is a single-shot command (one order per invocation) rather than
  an interactive REPL, to keep the argparse interface scriptable — this
  matches the "Accept and validate input via CLI" requirement directly.
- **About `logs/trading_bot.log`:** this repo ships with a sample log
  containing one MARKET and one LIMIT order as required by the task
  deliverables. Because placing real orders requires a personal testnet
  API key (which isn't available in the environment used to prepare this
  submission), the shipped log was generated by exercising the exact same
  `bot.orders.place_order()` code path with a stand-in client object that
  returns realistic Binance-shaped responses, so the log format, request
  parameters, and validation behavior are identical to a live run. Running
  the `python cli.py ...` commands above with real testnet credentials will
  append genuine live-order entries to the same file.

## Bonus Implemented

- **Third order type:** STOP_LIMIT (Binance `STOP` order), with its own
  validation rule requiring both `--price` and `--stop-price`.
